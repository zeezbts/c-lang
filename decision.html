Decision Making Statements

Decision making structures require the programmer to specify one or more conditions to be evaluated by the program.

START
Condition
TRUE (Code A)
FALSE (Code B)

1. Simple If

This is used when you only care if the condition is True. If it's False, the program just moves on to the next line.

1. Simple If Syntax

if (condition) {
  // code to be executed if condition is true
}

if (age >= 18) {
  printf("You are an adult.");
}

If the condition is false, the program does nothing and moves to the next line.

2. If-Else

This is used when you want to provide two different paths: one for True and one for False.

2. If-Else Syntax

if (condition) {
  // code if condition is true
}
else {
  // code if condition is false
}

if (age >= 18) {
  printf("Eligible to vote");
} else {
  printf("Not eligible");
}

This provides a specific plan for both True and False results.

3. Switch Statement

Definition: It allows a variable to be tested for equality against a list of values (cases).

syntax:
switch(expression) {
  case value1: //code; break;
  default: //code;
}

Full Example: Even or Odd

#include <stdio.h>
void main() {
  int num;
  printf("Enter a number: ");
  scanf("%d", &num);

  if (num % 2 == 0) {
    printf("%d is Even", num);
  } else {
    printf("%d is Odd", num);
  }
}

Nested If-Else: Largest of Three Numbers

Definition: A nested if-else is an if-else statement inside another if or else block. It is used when we need to check multiple conditions one after another.

#include <stdio.h>
void main() {
  int a, b, c;
  printf("Enter three numbers: ");
  scanf("%d %d %d", &a, &b, &c);

  if (a >= b) {
    if (a >= c)
      printf("%d is the largest", a);
    else
      printf("%d is the largest", c);
  } else {
    if (b >= c)
      printf("%d is the largest", b);
    else
      printf("%d is the largest", c);
  }
}
Logic Explained:
1. First, we compare a and b.
2. If a is bigger, we then compare a with c.
3. If b was bigger, we compare b with c.

Important Points to Remember

  • If Statement:
    • In C, any non-zero value is treated as TRUE, and zero is treated as FALSE.
    • If you only have one line of code after if, the curly braces { } are optional, but recommended for clarity.
  • If-Else Statement:
    • The else block cannot exist without a matching if.
    • It creates a "mutually exclusive" choice—only one of the two blocks will ever run.
  • Switch Statement:
    • The Break Keyword: If you forget break;, the program will "fall through" and execute all following cases until it hits a break or the end.
    • Constant Values: Case labels must be constants or literals (like 1, 2, or 'A'). You cannot use variables or expressions in case labels.
    • Default Case: The default block is optional but good practice; it runs if no case matches.
  • Nested If-Else:
    • Indentation: Always indent your code properly so you can see which else belongs to which if.
    • There is no limit to how many levels you can nest, but nesting too deep (more than 3 levels) makes code hard to read.
Common Mistake: Using a single = instead of double == in conditions.
Example: if(x = 5) will always be true because it assigns 5 to x. Use if(x == 5) to compare.
